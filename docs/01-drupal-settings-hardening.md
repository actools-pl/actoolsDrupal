## 1. Understanding the File Structure

Drupal's configuration hierarchy on your Actools server:

```
/home/actools/docroot/prod/
├── sites/
│   └── default/
│       ├── default.settings.php     ← Drupal's template (never edit)
│       ├── settings.php             ← Your active config (gitignored)
│       ├── default.services.yml     ← Services template
│       └── services.yml             ← Your active services (gitignored)
│           files/                   ← Public files (web-accessible)
│           private/                 ← Private files (NOT web-accessible)
```

**What Actools injects automatically into settings.php:**
- Database credentials (via `$databases` array)
- Trusted host patterns (via `$settings['trusted_host_patterns']`)
- S3FS credentials (via `$config['s3fs.settings']` using `getenv()`)
- Redis connection (if enabled)

**What you add manually:**
- Private file path
- Session hardening
- Production error suppression
- Performance tuning
- Environment-specific overrides

## 2. Public vs Private Files — When to Use Each

This is the most commonly misunderstood Drupal configuration decision.

### Public Files (`sites/default/files/`)

**Web-accessible via direct URL.** Anyone who knows or guesses the URL can download the file without logging in.

```
https://feesix.com/sites/default/files/annual-report.pdf
```

**Use public files for:**
- Images displayed on public pages (logos, hero images, article photos)
- Files intentionally available without login (brochures, public PDFs)
- CSS/JS aggregates (Drupal manages these automatically)
- Thumbnails and image derivatives

**Never use public files for:**
- Documents that require login to access
- User-uploaded content that may be sensitive
- PDFs with confidential or commercially sensitive information
- Any file where access should be controlled by Drupal permissions

### Private Files (`sites/default/private/` or a path OUTSIDE the web root)

**Not web-accessible.** All access goes through Drupal's access control system. Drupal checks permissions before serving the file.

```
# User tries to access a private file
https://feesix.com/system/files/private-report.pdf
# → Drupal checks if user has permission → serves or denies
```

**Use private files for:**
- PDF reports generated by XeLaTeX (your primary use case)
- Documents uploaded by authenticated users
- Invoice PDFs, contracts, confidential content
- Any file where access depends on user role or specific permissions

### For Your Content Generator (feesix.com)

Given that Actools is powering a content generator with XeLaTeX PDF output, **all generated PDFs should use private files**. Configure this immediately:

```php
// In settings.php — set private file path OUTSIDE the web root
$settings['file_private_path'] = '/home/actools/docroot/prod/private';
```

```bash
# Create the private directory and set correct permissions
mkdir -p /home/actools/docroot/prod/private
chmod 750 /home/actools/docroot/prod/private
chown www-data:www-data /home/actools/docroot/prod/private
```

Then in Drupal admin:  
**Admin → Configuration → Media → File system**  
Set "Private file system path" to `/home/actools/docroot/prod/private`

## 3. Hardened settings.php — Complete Example

This is a production-ready, hardened `settings.php` for your Actools stack. It replaces the auto-generated version.

```php
<?php

/**
 * Actools — Hardened settings.php
 * Drupal 11 · PHP 8.3 · Actools v11.0
 *
 * NEVER commit this file to git.
 * NEVER hardcode passwords here — use environment variables.
 */

// =============================================================================
// DATABASE — credentials come from Docker environment variables
// =============================================================================
$databases['default']['default'] = [
  'driver'    => 'mysql',
  'host'      => 'db',
  'port'      => '3306',
  'database'  => getenv('DB_NAME') ?: 'actools_prod',
  'username'  => getenv('DB_USER') ?: 'actools_prod',
  'password'  => getenv('DB_PASS') ?: '',
  'prefix'    => '',
  'charset'   => 'utf8mb4',
  'collation' => 'utf8mb4_general_ci',
  'pdo'       => [
    PDO::ATTR_TIMEOUT => 5,
    PDO::MYSQL_ATTR_INIT_COMMAND => "SET sql_mode='ANSI,TRADITIONAL'",
  ],
];

// =============================================================================
// TRUSTED HOST PATTERNS — injected by Actools installer
// =============================================================================
$settings['trusted_host_patterns'] = [
  '^feesix\.com$',
  '^.*\.feesix\.com$',
];

// =============================================================================
// HASH SALT — critical for security, auto-generated by Drupal installer
// =============================================================================
// $settings['hash_salt'] = ''; // Set by Drupal installer — do not remove

// =============================================================================
// FILE SYSTEM
// =============================================================================
// Public files — web accessible
$settings['file_public_path'] = 'sites/default/files';

// Private files — NOT web accessible, access controlled by Drupal
// All XeLaTeX-generated PDFs should be stored here
$settings['file_private_path'] = '/home/actools/docroot/prod/private';

// Temporary files
$settings['file_temp_path'] = '/tmp';

// =============================================================================
// REDIS CACHE — using Redis container on actools_net network
// =============================================================================
if (getenv('REDIS_HOST')) {
  $settings['redis.connection']['interface'] = 'PhpRedis';
  $settings['redis.connection']['host']      = getenv('REDIS_HOST') ?: 'actools_redis';
  $settings['redis.connection']['port']      = 6379;
  $settings['cache']['default'] = 'cache.backend.redis';
  $settings['cache']['bins']['form'] = 'cache.backend.database';
  $settings['container_yamls'][] = 'modules/contrib/redis/example.services.yml';
  $settings['container_yamls'][] = 'modules/contrib/redis/redis.services.yml';
}

// =============================================================================
// S3FS — credentials via environment variables, injected by Actools
// =============================================================================
$config['s3fs.settings']['access_key']         = getenv('AWS_ACCESS_KEY_ID') ?: '';
$config['s3fs.settings']['secret_key']         = getenv('AWS_SECRET_ACCESS_KEY') ?: '';
$config['s3fs.settings']['bucket']             = getenv('S3_BUCKET') ?: '';
$config['s3fs.settings']['region']             = getenv('AWS_REGION') ?: 'us-east-1';
$config['s3fs.settings']['use_s3_for_public']  = TRUE;
$config['s3fs.settings']['use_s3_for_private'] = FALSE; // Keep PDFs in private, not S3

$_s3_endpoint = getenv('S3_ENDPOINT_URL');
if (!empty($_s3_endpoint)) {
  $config['s3fs.settings']['use_customhost'] = TRUE;
  $config['s3fs.settings']['hostname']       = $_s3_endpoint;
}

$_cdn_host = getenv('ASSET_CDN_HOST');
if (!empty($_cdn_host)) {
  $config['s3fs.settings']['use_cname'] = TRUE;
  $config['s3fs.settings']['domain']    = $_cdn_host;
}

// =============================================================================
// SESSION SECURITY
// =============================================================================
// Increase session cookie lifetime (default is browser session)
$settings['session_write_interval'] = 180;

ini_set('session.cookie_lifetime', 2000000);
ini_set('session.gc_maxlifetime', 2000000);

// Secure cookie settings (HTTPS only)
ini_set('session.cookie_secure', TRUE);
ini_set('session.cookie_httponly', TRUE);
ini_set('session.cookie_samesite', 'Strict');
ini_set('session.use_strict_mode', TRUE);

// =============================================================================
// PRODUCTION ERROR HANDLING
// =============================================================================
// Never display errors to users in production
$config['system.logging']['error_level'] = 'hide';

// Disable Drupal's error display
$settings['rebuild_access'] = FALSE;

// Suppress PHP errors from browser output
ini_set('display_errors', FALSE);
ini_set('log_errors', TRUE);
ini_set('error_log', '/var/log/php/drupal-error.log');

// =============================================================================
// PERFORMANCE
// =============================================================================
// CSS and JS aggregation
$config['system.performance']['css']['preprocess'] = TRUE;
$config['system.performance']['js']['preprocess']  = TRUE;

// OPcache — validate_timestamps=0 for max performance (set in actools.env)
// PHP_OPCACHE_VALIDATE_TIMESTAMPS=0

// Disable automatic cron (use drush cron via system cron instead)
$config['automated_cron.settings']['interval'] = 0;

// =============================================================================
// SECURITY HARDENING
// =============================================================================
// Disable Drupal's built-in PHP filter (security risk)
$settings['php_storage']['twig']['directory'] = 'sites/default/php';

// Restrict config sync directory
$settings['config_sync_directory'] = '../config/sync';

// Prevent information disclosure
$settings['sa_core_2014_005_redirect_allow_regexp'] = FALSE;

// =============================================================================
// ENVIRONMENT DETECTION
// =============================================================================
// Detect environment from Docker env var (set in docker-compose.yml)
$environment = getenv('APP_ENV') ?: 'production';

if ($environment === 'development') {
  // Development overrides — never in production
  $settings['cache']['default'] = 'cache.backend.null';
  $config['system.performance']['css']['preprocess'] = FALSE;
  $config['system.performance']['js']['preprocess']  = FALSE;
  $config['system.logging']['error_level'] = 'verbose';
  ini_set('display_errors', TRUE);
}
```

## 4. Database Configuration Hardening

### Add a Read-Only User for Reporting

```sql
-- Run inside the db container
docker compose exec -T db mariadb -uroot -p"${DB_ROOT_PASS}" << SQL
CREATE USER IF NOT EXISTS 'actools_readonly'@'%' IDENTIFIED BY 'strong_password_here';
GRANT SELECT ON actools_prod.* TO 'actools_readonly'@'%';
FLUSH PRIVILEGES;
SQL
```

### Connection Pool Settings

Add to `settings.php` for high-traffic sites:

```php
$databases['default']['default']['pdo'] = [
  PDO::ATTR_TIMEOUT         => 5,
  PDO::ATTR_PERSISTENT      => FALSE,
  PDO::MYSQL_ATTR_COMPRESS  => TRUE,
];
```

### MariaDB SSL Connection (Future Phase 5)

```php
$databases['default']['default']['pdo'] = [
  PDO::MYSQL_ATTR_SSL_CA   => '/etc/ssl/certs/ca-certificates.crt',
  PDO::MYSQL_ATTR_SSL_VERIFY_SERVER_CERT => TRUE,
];
```

## 5. Trusted Host Patterns

Trusted host patterns prevent HTTP host header injection attacks.

### Basic Setup (already injected by Actools)

```php
$settings['trusted_host_patterns'] = [
  '^feesix\.com$',
  '^.*\.feesix\.com$',   // All subdomains including preview environments
];
```

### With Load Balancer / Reverse Proxy

If you add Cloudflare or a load balancer in Phase 5:

```php
// Trust Cloudflare's forwarded IPs
$settings['reverse_proxy'] = TRUE;
$settings['reverse_proxy_addresses'] = [
  '103.21.244.0/22',
  '103.22.200.0/22',
  '103.31.4.0/22',
  '104.16.0.0/13',
  '104.24.0.0/14',
  // Add Cloudflare IP ranges
];
$settings['reverse_proxy_trusted_headers'] = 
  \Symfony\Component\HttpFoundation\Request::HEADER_X_FORWARDED_FOR |
  \Symfony\Component\HttpFoundation\Request::HEADER_X_FORWARDED_HOST |
  \Symfony\Component\HttpFoundation\Request::HEADER_X_FORWARDED_PORT |
  \Symfony\Component\HttpFoundation\Request::HEADER_X_FORWARDED_PROTO;
```

## 6. Redis Cache Configuration

```bash

docker compose exec -T php_prod bash -c "
  cd /var/www/html/prod
  composer require drupal/redis
  ./vendor/bin/drush en redis --yes
  ./vendor/bin/drush cr
```

### Cache Bin Configuration

```php
// In settings.php
$settings['cache']['default'] = 'cache.backend.redis';

// Keep these in database (not Redis) — they need persistence
$settings['cache']['bins']['form']       = 'cache.backend.database';
$settings['cache']['bins']['discovery']  = 'cache.backend.database';

$settings['cache']['bins']['flood'] = 'cache.backend.database';
```

## 7. S3 / File Storage Configuration

### When to Use S3 for Public vs Private

```php
// For a content generator site:
$config['s3fs.settings']['use_s3_for_public']  = TRUE;   // Images, CSS, JS
$config['s3fs.settings']['use_s3_for_private'] = FALSE;  // PDFs stay local/private
```

**Rationale:** Public assets (images, CSS) benefit from S3 + CDN delivery worldwide. Private PDFs generated by XeLaTeX should remain on the server in the private file system where Drupal controls access — sending them to S3 adds latency and requires signed URL generation for each download.

### S3 Path Structure

```
s3://your-bucket/
├── public/           ← Drupal public files
│   ├── styles/       ← Image derivatives
│   └── ...
└── preview/          ← Actools preview environment files
    ├── feature-123/
    └── ...
```

## 8. Session Security

### Harden Cookie Settings

```php
// settings.php
ini_set('session.cookie_secure', TRUE);     // HTTPS only
ini_set('session.cookie_httponly', TRUE);   // No JavaScript access
ini_set('session.cookie_samesite', 'Strict'); // CSRF protection
ini_set('session.use_strict_mode', TRUE);   // Reject uninitialized session IDs
ini_set('session.use_only_cookies', TRUE);  // No session ID in URL
```

### Session Lifetime for Admin Users

```php
// Shorter admin sessions (30 minutes)
$config['system.session']['cookie_lifetime'] = 1800;
```

## 9. Error Handling in Production

### Three-Level Approach

**Level 1 — PHP level** (suppress from browser):
```php
ini_set('display_errors', FALSE);
ini_set('log_errors', TRUE);
ini_set('error_log', '/var/log/php/drupal-error.log');
```

**Level 2 — Drupal level** (hide from page output):
```php
$config['system.logging']['error_level'] = 'hide';
```

**Level 3 — Caddy level** (custom error pages in Caddyfile):
```
handle_errors {
    respond "{http.error.status_code} {http.error.status_text}"
}
```

### Check Logs

```bash
# PHP errors
docker compose exec php_prod tail -f /var/log/php/drupal-error.log

# Or via Actools CLI
actools slow-log prod
```

## 10. Environment-Specific Overrides

Add to `docker-compose.yml` environment section:

```yaml
php_prod:
  environment:
    APP_ENV: "production"
    DB_NAME: "actools_prod"
    DB_USER: "actools_prod"
    DB_PASS: "${DB_PROD_PASS}"
    REDIS_HOST: "actools_redis"
```

For preview environments, Actools automatically appends to `settings.php`:

```php
// Preview environment override — feature-123
$databases['default']['default']['database'] = 'actools_pr_feature_123';
$databases['default']['default']['username'] = 'actools_pr_feature_123';
$databases['default']['default']['password'] = 'generated_pass';
$settings['trusted_host_patterns'][] = '^feature-123\.feesix\.com$';
```

## 11. Permissions Checklist

Run this after any settings.php change:

```bash
# Correct permissions for settings.php (read-only to web server)
chmod 440 /home/actools/docroot/prod/sites/default/settings.php
chown www-data:www-data /home/actools/docroot/prod/sites/default/settings.php

# Correct permissions for files directory
chmod 755 /home/actools/docroot/prod/sites/default/files
chown -R www-data:www-data /home/actools/docroot/prod/sites/default/files

# Correct permissions for private directory
chmod 750 /home/actools/docroot/prod/private
chown -R www-data:www-data /home/actools/docroot/prod/private

# Verify with Drupal's built-in check
docker compose exec -T php_prod bash -c \
  "cd /var/www/html/prod && ./vendor/bin/drush status"
```

```bash
"
```
